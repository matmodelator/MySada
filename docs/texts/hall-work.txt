Вот структура hall-work.html для варианта только зал, без кода.

1. Верхняя панель

То, что видно сразу.

Состав:

название смены / зала
дата и время
фильтр по зонам
легенда статусов
кнопка обновить
кнопка открыть редактор схемы

Пример логики:

Hall Work
Zone: All / Hall / VIP / Pool / Sauna / Bar / Terrace
2. Центральная часть — сам план зала

Это главный экран.

Он берет данные из:

tables-layout.json

И показывает:

столы на своих местах
номера столов
цвет по статусу
тип стола
зону

То есть редактора тут уже нет.

Нельзя:

таскать столы
менять тип
менять координаты

Можно только работать со статусами и данными.

3. Карточка выбранного стола

Когда нажимаешь на стол — справа или снизу открывается блок управления этим столом.

Что в карточке:

Table 12
Zone: VIP
Type: square
Seats: 4
Status: occupied
Guests: 3
Waiter: Alex
Note: birthday
4. Кнопки действий по столу

Минимальный набор:

Seat guests
Reserve
Set cleaning
Free table
Open bill

Для “только зал” можно даже проще:

Occupied
Reserved
Cleaning
Free

То есть кнопки просто меняют статус.

5. Поля по столу

Минимум для MVP:

guests — сколько гостей
waiter — кто ведет стол
note — заметка
status

Этого уже хватает для живой работы.

6. Хранилище состояния

Отдельно от layout.

Например:

tables-state.json

Он хранит только текущее состояние работы.

То есть:

layout
стол №7 стоит тут
тип square
зона hall
state
стол №7 занят
2 гостя
официант Max
заметка VIP guest
7. Слияние двух источников

При открытии hall-work.html система делает так:

1. читает tables-layout.json
2. читает tables-state.json
3. объединяет по номеру стола
4. показывает готовый рабочий зал
8. Цветовая логика

Очень важна, потому что это главный быстрый ориентир.

Например:

free      → green
occupied  → red
reserved  → yellow
cleaning  → gray
bill      → blue

Если без bill, то можно оставить 4 цвета.

9. Правая рабочая панель

Лучше сделать справа.

Там:

карточка выбранного стола
редактирование гостей
официант
заметка
смена статуса
кнопка сохранить

Это удобнее, чем popup по центру.

10. Нижняя сервисная панель

Не обязательно, но полезно.

Можно вывести:

Free: 12
Occupied: 8
Reserved: 3
Cleaning: 1
Total: 24

Это уже почти маленькая диспетчерская.

11. Фильтр по зонам

Полезен сразу.

Например:

All
Hall
Top Terrace
Bottom Terrace
Left Terrace
Right Terrace
VIP
Pool
Sauna
Bar

При фильтре можно:

либо скрывать остальные столы
либо затемнять их
12. Что НЕ нужно на первом шаге

Чтобы не раздувать MVP, пока не нужны:

кухня
касса
склад
блюда
техкарты
чеки
клиенты как база
история заказов

Для тебя сейчас важен именно оперативный зал.

13. Минимальный сценарий работы

Вот как это должно жить:

1. Открыли hall-work.html
2. Загрузился план
3. Видим свободные и занятые столы
4. Клик по столу
5. Выбрали статус
6. Ввели гостей / официанта / заметку
7. Сохранили

Это уже реальный MVP для зала.

14. Самая правильная структура страницы
[ Top bar ]

[ Floor plan ]     [ Selected table panel ]

[ Summary line ]

То есть:

сверху управление
слева/в центре план
справа карточка стола
снизу итоги
15. Что делать следующим шагом

После этой структуры следующий разумный шаг:

1. описать JSON state
2. описать UI карточки стола
3. потом уже писать hall-work.html

Если идти правильно, то сначала надо определить формат состояния стола, а не сразу лезть в код.
